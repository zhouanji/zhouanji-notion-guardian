块元素 & 内联元素

块（block）元素：

显示效果：显示时前后各有一个换行

使用时机：构建页面初期自顶向下进行分块

<h> <ul> <li> <blockquote>

内联（inline）元素：

显示效果：显示在页面文本流的行内

使用时机：完善页面时在块内部加入

<img> <em> <q>

有些元素既是也是： <a>

有些元素既不是也不是： <br>

正常元素 & Void 元素

正常元素：必须包含内容的元素 —— <开始标签> + 内容 + <结束标签>

Void 元素：没有任何内容的元素 —— <开始标签>

<br> <img>

正常元素和Void元素之间没有交集！不存在既是A又是B的元素！

id 属性

元素的唯一标识（命名需要确保在页面内唯一）

命名区分大小写

可以作为 <a> 元素 href 属性 链接的锚点

可以作为 CSS 选择器的对象

title 属性

对 元素内容 or 元素点击行为 的文本描述

浏览器的处理方式：鼠标悬浮展示信息；视障人士辅助 等等

class 属性

为元素实例进行分组命名，方便 CSS 选择器进行多个元素的选择

不同元素可以属于相同类 <元素名1 class=”类名”> <元素名2 class=”类名”>

相同元素可以属于不同类 <元素名 class=”类名1 类名2 类名3 类名4 ……”>

href 属性（超文本引用 hyper text reference）：各类资源 URL 地址

可以使用 # 直接链接到页面中有 id 属性 的 元素，格式为 页面文件URL#id (”www.xxxx.com/xx/index.html#id”) 或 页面目录URL/#id (”www.xxxx.com/xx/#id”)

target 属性：指定打开链接资源的 浏览器窗口/标签页 名称

默认 → 始终在本页打开资源

“_blank” → 始终打开新的窗口/标签页

“自定义命名” → 有相同目标名的链接将会在同一个窗口/标签页打开

元素内容即隐性规定了链接点击范围

<img> 图像

src 属性：图片 URL 地址（放置网页地址无法显示图片，点击也无法实现跳转）

alt 属性：图片加载失败时，显示的对图片的文本描述；视障人士辅助

width height 属性：指定图片的宽高像素，用于浏览器提前建立页面布局结构

图像获取和解析流程：

  1. 浏览器首先根据URL向对应服务器请求页面html，服务器响应html
  1. 浏览器解析html获得页面中的图像URL，向对应服务器同时请求多个图像资源
  1. 获取具体图像资源前，如果html中图像指定了width和height，则浏览器可以提前建立页面布局结构，若未指定，则获得图像后浏览器需根据图像实际大小重新调整页面布局
  1. 获得具体图像资源后，浏览器将根据width和height属性 或 浏览器中设置了”自动调整图像大小”选项 重新调整大小（注意此时已按照服务器端的图像实际大小下载到本地）

图像格式：

  • JPEG：适合照片和复杂图像（1600万色）；有损；文件小（此外还可以调整图片质量进一步控制文件大小）；
  • PNG：适合LOGO/单色/几何（PNG-24 百万色，PNG-16 千色，PNG-8 256色）；支持透明；无损；文件大小取决于颜色数（相同颜色数PNG的压缩要优于GIF）；
  • GIF：适合动画（256色）；半支持透明；无损；支持动画；文件大；

默认页面

URL地址为一个服务器端的目录时，服务器收到请求，会从目录中寻找到默认页面并返回

不同Web服务器规定了不同的默认页面文件名称，如 “index.html” ”default.htm” “index.php”

因此，将网站发布给他人时，最好只提供主页目录的URL（e.g. “www.baidu.com”），而非精确到具体默认页面名称（e.g. “www.baidu.com/index.html“），因为部署的服务器可能发生更换，相应的默认页面名称也需要随之变化，原来发布的带页面名称的URL就会失效
