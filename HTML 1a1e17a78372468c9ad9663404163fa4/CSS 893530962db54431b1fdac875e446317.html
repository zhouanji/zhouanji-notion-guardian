外部样式表（.css）

将 CSS 样式单独提出来作为一个独立文件，可以方便不同页面复用相同的样式规则

W3C 验证工具（CSS标准）：http://jigsaw.w3.org/css-validator/

混合样式表（一个HTML同时使用多个CSS）

用途 1：样式表之间为串联关系（引入顺序靠后的样式表覆盖靠前的冲突规则），由粗粒度样式表到细粒度样式表，方便同一站点的多个子页面进行复用，实现在保持相似主题的基础上进行个性化调整

<link … href=”基础样式.css” …> <link … href=”个性化样式.css” …> <link … href=”细粒度调整.css” …>

用途 2：样式表之间为并联关系（浏览器使用媒体查询选择其中一套样式规则进行使用），旨在面向不同设备提供不同的样式规则

实现方式 A：在 HTML 文件 中引入面向不同设备的多个 CSS 文件，并分别在 <link> 标签中使用 media 属性 增加媒体查询，浏览器将通过媒体查询选择其中的一个 CSS 文件进行加载：

<link … href=”PC端样式.css” … media=“aaaa”> <link … href=”移动端样式.css” … media=”bbbb”> <link … href=”打印机样式.css” … media=”cccc”>

实现方式 B：在一个 CSS 文件 中创建面向不同设备的多组规则，并使用 @media 标记，浏览器通过媒体查询在标记 @media 的规则中选择一组加载，并加载所有其他普通规则：

@media aaaa { 面向 满足aaaa设备 的规则集合, … … } @media bbbb { 面向 满足bbbb设备 的规则集合, … … } @media cccc { 面向 满足cccc设备 的规则集合, … … } 其他普通规则 /* 普通规则适用于所有设备 */

媒体查询 的 可选属性值：参考 http://www.w3.org/TR/css3-mediaqueries/

规则

将 不同元素的相同样式（属性+值） 归在一起，编写为一条规则

其中元素如有对其他属性的额外样式需求，则单独编写规则

选择器（元素实例的集合）

示例代码

选择单个元素： h1

选择多个元素： h1, h2

选择某个类中的某个元素： h1.classname

选择某个类中的所有元素： .classname

选择某个 id 类型不限的元素： #idname

选择某个 id 特定类型的元素： h1#idname

迭代选择子孙元素： 父元素 子孙元素 （迭代选择父元素节点下的所有匹配的子孙元素）

直接选择孩子元素： 父元素>孩子元素 （选择父元素节点的直接孩子中匹配的孩子元素）

按多层路径限制选择子孙元素： 父元素 子元素 孙元素 （选择父节点的某子节点的孙节点）

伪类（元素状态）： a:visited

顺序靠后的样式生效

伪类

元素可能拥有不同 状态（未访问 link / 已访问 visited / 鼠标悬浮 hover / 焦点 focus / 活动 active 等等），每个状态为一个伪类，可以对伪类指定 CSS 样式，浏览器会检查不同状态的元素并将其放入不同的伪类，从而应用不同伪类的 CSS 样式

元素可能同时处于多种状态，所以 伪类规则的顺序 很重要：

e.g. <a> 可能同时处于 已访问 & 悬浮，所以伪类规则的顺序最好为 link, visited, hover, focus, active，来保持不同状态间显示样式的优先级

浏览器对某个特定元素特定属性 CSS 规则的解析流程

  1. 收集所有样式表：Web页面作者的样式表、用户自行配置的样式表（部分浏览器允许用户定义自己的样式表）、浏览器的默认样式表
  1. 遍历选择器和页面结构，找到该元素对应属性的所有规则
  1. 优先级原则一：页面作者 → 用户 → 浏览器

    读者如果一定要覆盖页面作者的样式，可以在规则末尾添加 !important 标记，例如： font-size: 200% !important;

  1. 优先级原则二：特定性越高，优先级越高

    选择器的特定性计算公式： 特定性 = id数量 x 100 + 类或伪类数量 x 10 + 元素数量 x 1

    选择器中逗号分隔部分视为不同的规则，分别计算特定性（例如 h1, h2 视为两条规则 ）

  1. 优先级原则三：样式表中越靠后的规则，优先级越高

    若 HTML 引入了多个 CSS 文件，则不同 CSS 中的规则顺序以 HTML 的引用顺序为准

规则缩写

针对某个特定对象（background / font / margin / border 等）的不同维度的属性（即使用横线 - 分隔的属性）可以合并为一条规则进行缩写：

由 属性值 可以区分 属性维度 的，属性值顺序任意

xx-width: thin; xx-style: solid; — 缩写 —> xx: solid thin #007e7e; xx-color: #007e7e;

由 属性值 无法区分 属性维度 的，属性值顺序固定

padding-top: 0px; padding-right: 20px; — 缩写 —> padding: 0px 20px 10px 30px; padding-bottom: 10px; padding-left: 30px;

字体规则的缩写

font: [font-style] [font-variant] [font-weight] font-size[/line-height] font-family;

盒模型

盒模型是CSS看待元素的方式：CSS将 每个元素（包括内联元素） 都视为一个盒子

内容区（content area）

内容区： 元素的内容（文本或图像），盒子大小正好能包含所有内容

宽度 width 默认值 auto 占满整个浏览器宽度； 百分数 相对元素所在容器宽度(<div><body>等)

高度 height 通常不修改！默认值 auto 将按内容多少自动调整高度，指定高度不足内容将会溢出

边框（border）

边框： 内边距外框边线，边框内部都可以认为是元素本身

样式 border-style solid double groove ouset inset dashed dotted ridge

宽度 border-width thin medium thick 像素值

颜色 border-color 属性值见下文颜色章节编码

某侧 border-边位置-属性 如 border-top-color border-left-style 等等

圆角 border-角位置-radius 如 border-top-left-radius border-bottom-right-radius 等等

内边距（padding）

透明，旨在为元素内容周围增加额外空间

外边距（margin）

透明，旨在提供元素之间的额外空间

padding / margin 指定四周边距宽度 padding-left margin-right 等 指定单独某侧边距宽度

注意属性样式顺序：通常先整体指定四周宽度，后特殊指定某侧宽度进行覆盖

padding: 25px; padding-left: 80px;

内边距 & 外边距 只能修改四周大小，不能修改样式

元素的 背景颜色 & 背景图片 会延申到 内边距，不延申到 外边距

背景颜色 & 背景图像

背景颜色 background-color

背景图像 background-image 属性值格式为 url(xxxx/xx/x.x)

图像重复 background-repeat 属性值 no-repeat(不) repeat-x(x轴) repeat-y(y轴) inherit(继承)

图像位置 background-position 属性值 top bottom left right center 等等

二者会在最底图层覆盖 内容区 + 内边距，不覆盖 外边距

<div> 与 <span>

二者均用于对元素进行逻辑划分，均可以嵌套使用

可以设置 id 和 class 属性并应用于 CSS 选择器

对 <div> 或 <span> 指定的规则中，有些属性可以向下继承，有些属性不可以向下继承

<div>

对 块元素 进行逻辑划分

<span>

对 内联元素 进行逻辑划分

字体属性 font-family

属性值

值域： 字体名 or 字体系列名（字体系列：每个字体系列都包含大量字体）

示例： font-family: 字体1, 字体2, 字体3, 字体4, …, 字体系列;

候选字体列表（即属性值）：

每个浏览器和操作系统自带的字体库不同，浏览器将按顺序逐一尝试属性值中各个字体直至找到支持的字体，故通常前面几个字体根据显示效果优先级排序，并尽量覆盖到多种操作系统和浏览器，最后一个位置放置通用字体系列来保底

通常属性值（即候选字体列表）都来自同一个字体系列

字体名区分大小写！

英文字体系列：

  • Sans-serif 无衬线
  • Serif 有衬线
  • Monospace 等宽字符
  • Cursive 手写
  • Fantasy 趣味

引入 Web 字体

字体引入流程：

  1. 寻找字体资源，并下载字体的不同文件格式（不同浏览器支持不同字体文件格式）
  1. 将所有格式的字体文件放在用户能访问到的某台服务器上，记录URL
  1. 使用CSS内置规则 @font-face 来在 .css 文件中引入：给字体命名 并 提供字体文件URL
  1. 在 .css 文件中 添加 CSS 样式规则： font-family 属性值

用户访问页面时浏览器的处理过程：

  1. 浏览器向服务器请求 HTML 和 CSS 文件
  1. 浏览器发现页面引入了 Web 字体，按顺序尝试 src 中各个URL的字体文件格式，并向服务器请求对应字体文件进行加载

缺点：

  • 获取和加载 Web字体 比较耗时
  • 移动设备、小型设备 并不支持 Web字体

大小属性 font-size

属性值

px：像素

%：百分比（相对父元素的字体大小）

em：倍数（相对父元素的字体大小）（% 与 em 等价）

关键字：浏览器将关键字映射为像素值，不同浏览器的映射表不同 xx-small, x-small, small, medium, large, x-large, xx-large

( 特殊的，对于 <body> 元素使用 % 或 em 定义字体大小，则为相对浏览器的默认字体大小 )

字体大小设置的方法论

  1. 使用 关键字（small 或 medium） 设置 <body> 元素
  1. 使用 em 或 % 设置 其他元素

由此，页面开发者 或 用户 可以方便地整体性地调整字体大小

风格样式类属性

粗细属性 font-weight

bold（粗体） normal（正常）

bolder lighter （相对继承的文本样式）

100 的倍数 （100-900）

行间距属性 line-height

像素

em / 百分数（相对当前元素节点字体大小的倍数，但子孙元素将会继承这个行高的像素绝对值）

数值 （子孙元素继承时 设置行高为 相对子孙元素各自字体大小的倍数）

风格属性 font-style

italic（字体本身提供的斜体风格） oblique（浏览器将正常文字倾斜）

装饰属性 text-decoration

line-through（删除线） underline（下划线） overline（上划线） none（取消装饰）

属性值用空格分隔，可以同时对文本加入多种装饰

颜色

颜色编码方式

颜色名： 16种基本颜色，150种扩展颜色 （颜色名不区分大小写）

红绿蓝-百分比： rgb(xx%, xx%, xx%)

红绿蓝-数值： rgb(xxx, xxx, xxx) （数值范围0-255）

红绿蓝-十六进制： #xxxxxx （每两位十六进制数为一种颜色的数值）

简写规则：十六进制编码中，如果每组分量的两位数字全部相同，则可以使用简写，例如 #ccbb00 -> #cb0，#ccbb10 则不能简写

( 通过后三种红绿蓝混合的方式可以得到超过1600万种颜色 )

布局

内容对齐属性 text-align

使用对象： 只能对 块元素 使用该属性

作用效果： 该 块元素 迭代包含的所有下属 内联元素 均应用此对齐方式 （不限于文本）
